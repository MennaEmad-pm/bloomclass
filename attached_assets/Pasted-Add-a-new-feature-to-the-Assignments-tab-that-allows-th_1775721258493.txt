Add a new feature to the Assignments tab that allows the admin to post assignment
requirements. These posts appear in the same assignments list but are visually
distinct from student submissions.

---

## 🗄️ SUPABASE SCHEMA UPDATE

Add a new table:

assignment_posts:
  - id (uuid, default: gen_random_uuid())
  - title (text, not null)
  - description (text, not null)
  - posted_at (timestamp, default: now())
  - posted_by (uuid, FK → auth.users)

RLS Policies:
  - All authenticated users can read assignment_posts
  - Only admin (is_admin = true in profiles) can insert/update/delete

---

## 📱 ASSIGNMENTS TAB CHANGES

### Admin View — New "Post Assignment" Button

At the top of the Assignments tab, if the logged-in user is admin, show a
prominent button:
  Label: "+ Post New Assignment"
  Style: filled button using primary color (#C9748A), full width, bold label

Tapping it opens a bottom sheet modal or full modal screen with:
  - Field 1: Assignment Name (text input, required)
  - Field 2: Assignment Description (multiline text input, min 4 lines, required)
  - "Post Assignment" submit button
  - "Cancel" button
  - Validate both fields before submitting
  - On success: dismiss modal, show toast "Assignment posted!", refresh list

---

## 📋 UNIFIED ASSIGNMENTS LIST

Merge both assignment_posts (from admin) and assignments (from students) into a
single chronological list, sorted by date descending (newest first).

Each item in the list is one of two card types:

---

---

### ⬜ Card Type B — Student Submission (existing style, no changes)

Keep the existing student submission card style:

## 🔀 MERGING THE TWO LISTS

Fetch both tables from Supabase in parallel using Promise.all():

  const [postsResult, submissionsResult] = await Promise.all([
    supabase.from('assignment_posts').select('*').order('posted_at', { ascending: false }),
    supabase.from('assignments').select('*').order('submitted_at', { ascending: false })
  ]);

Map each item to a unified shape with a "type" field:
  - type: 'admin_post' → from assignment_posts, use posted_at for sorting
  - type: 'student_submission' → from assignments, use submitted_at for sorting

Merge into one array and sort by date descending before rendering.

Use a renderItem function that checks item.type and returns the correct card
component (AdminPostCard or StudentSubmissionCard).
