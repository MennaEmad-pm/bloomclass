You are building a mobile application called "Course Companion" using React Native with Expo,
deployable via Expo Go. The project must be set up and runnable on Replit.

---

## ⚙️ TECHNICAL SETUP

- Framework: React Native with Expo (managed workflow)
- Initialize with: npx create-expo-app@latest CourseCompanion
- Navigation: Expo Router (file-based routing) OR React Navigation (stack + tab)
- Backend/Database: Supabase (auth + database + realtime)
- Styling: React Native StyleSheet (no Tailwind — not supported in RN)
- Icons: @expo/vector-icons (Ionicons)
- Notifications/Toasts: react-native-toast-message
- Forms: controlled components with useState
- Environment variables: store Supabase URL and anon key in app.config.js or .env

Make sure the app runs with "expo start" and can be scanned via Expo Go on a physical device.

## 👤 USER ROLES

### Student
- Sign up with email (unique) + password
- Profile: Full Name, LinkedIn URL, Current Position (default: "Student")
- Can view all course content
- Can submit assignments

### Admin (Menna Emad)
- Identified by isAdmin: true flag in Supabase "profiles" table
- Can edit each session's title, presentation link, and notes
- Admin controls hidden from regular students

---

## 🗄️ SUPABASE SCHEMA

Create these tables:

profiles:
  - id (uuid, FK → auth.users)
  - email (text)
  - full_name (text)
  - linkedin_url (text)
  - current_position (text, default: 'Student')
  - is_admin (boolean, default: false)

sessions:
  - id (int, 1–5)
  - title (text)
  - presentation_url (text, nullable)
  - notes (text, nullable)

assignments:
  - id (uuid)
  - student_name (text)
  - group_name (text)
  - task_link (text)
  - submitted_at (timestamp, default: now())

Enable Row Level Security (RLS):
  - Students can read all sessions and assignments
  - Only admin can update sessions
  - Any authenticated user can insert assignments
  - Users can only update their own profile
